# F9: 模型蒸馏与微调 — 从原理到实操

> **一句话：** 大模型太贵太慢，蒸馏+微调让小模型获得大模型的能力，跑在你的 3090 上。

---

## ⚡ 3 分钟速读版

```
一句话: 蒸馏=大模型教小模型, 微调=用少量数据改变模型行为, 二者可组合
3 个关键概念: 知识蒸馏(KD) / LoRA微调 / 量化(Quantization)
对 Muse 最重要的: 本地部署小模型可替代部分 API 调用，降低成本+延迟
面试必记: "蒸馏不是压缩参数，是让小模型学大模型的输出分布"
```

---

## 🔬 核心原理（三栏表）

| 概念 | 能力来源 | 激活方式 | 类比（仅类比） |
|------|---------|---------|--------------|
| **知识蒸馏 (KD)** | 大模型（Teacher）的输出概率分布包含"暗知识" | 训练小模型（Student）去模仿 Teacher 的 soft logits，而不是 hard labels | 仅类比：像老师教学生解题思路，而不只是给答案 |
| **LoRA** | 冻结原始参数，只训练两个小矩阵 A、B（低秩分解） | `W' = W + A×B`，A/B 的参数量远小于 W | 仅类比：像给模型"打补丁"而不是"重写" |
| **量化 (Quantization)** | 把 FP16/FP32 参数转成 INT8/INT4 | 推理时用低精度计算，精度换速度 | 仅类比：像把高清图压成缩略图，主体信息保留 |

## 1. 知识蒸馏（Knowledge Distillation）

### 1.1 为什么不直接用小模型训练？

```
直接用数据训练小模型:
  数据: "法国的首都是 [巴黎]"     ← Hard Label (只有正确答案)
  小模型学到: P(巴黎) = 1.0, P(其他) = 0.0

蒸馏训练:
  Teacher 输出: P(巴黎) = 0.85, P(里昂) = 0.08, P(马赛) = 0.05, P(伦敦) = 0.001
  小模型学到: "巴黎是首都，但里昂和马赛也是法国重要城市，伦敦不是"
                              ↑ 这些概率比例包含了 Teacher 的"暗知识"
```

> [!IMPORTANT]
> **蒸馏的核心不是"压缩参数"，而是"传递暗知识"。** Teacher 的 soft probabilities 包含了标签之间的关系（比如"里昂和巴黎都是法国城市"），这些信息在 hard label 里是看不到的。

### 1.2 蒸馏公式（简化版）

```
Loss = α × KL(Teacher_soft ‖ Student_soft) + (1-α) × CrossEntropy(true_label, Student_hard)
              ↑ 学 Teacher 的输出分布                   ↑ 也不忘正确答案

Temperature T: 在 softmax 里除以 T
  T=1: 正常概率分布
  T=5: 更"软"的分布 → 凸显差异 → 暗知识更明显
```

### 1.3 DeepSeek R1 的推理蒸馏

你提到的 `qwen3.5-27b-claude-4.6-opus-reasoning-distilled` 就是这种：

```
Teacher: Claude 4.6 Opus（超大模型，推理能力极强）
Student: Qwen 3.5 27B（中等模型，能跑在 3090 上）

蒸馏方式（Reasoning Distillation）:
  1. 用 Teacher 生成大量 <think>推理过程</think> + 答案
  2. 用这些数据 SFT 训练 Student
  3. Student 学会了 Teacher 的推理模式（不只是答案）

结果: 27B 模型获得了接近 Opus 级别的推理能力
```

> [!IMPORTANT]
> **推理蒸馏 vs 普通蒸馏：** 普通蒸馏只学 Teacher 的最终输出分布；推理蒸馏学的是 Teacher 的 **思考过程**（CoT），这能让 Student 获得 Teacher 的推理能力，而不只是记忆答案。

## 2. 微调（Fine-Tuning）

### 2.1 全参数微调 vs LoRA

| 方式 | 训练哪些参数 | 显存需求 | 效果 | 适用场景 |
|------|-----------|---------|------|---------|
| **全参数微调** | 所有参数 | 巨大（7B 需要 ~28GB） | 最好 | 有大量 GPU + 数据 |
| **LoRA** | 只训练 A/B 矩阵（<1% 参数） | 小（7B 只需 ~8GB） | 接近全参数 | 单卡 3090 可行 |
| **QLoRA** | LoRA + 4bit 量化基座 | 更小（7B 只需 ~6GB） | 略低于 LoRA | 显存极度受限 |

### 2.2 LoRA 原理

```
原始权重矩阵 W: [4096 × 4096] = 16M 参数

LoRA 低秩分解:
  A: [4096 × r]   r=16 时只有 65K 参数
  B: [r × 4096]   r=16 时只有 65K 参数

推理时: W' = W + A × B
  W 冻结（不训练）
  只训练 A 和 B → 总参数量: 130K vs 16M = 0.8%

为什么能工作？
  模型的"能力变化"通常是低秩的 — 
  改变模型的"行为模式"不需要改变每个参数，
  只需要在权重空间中做一个小的方向调整。
```

### 2.3 DPO vs RLHF

| 方式 | 做法 | 优势 | 劣势 |
|------|------|------|------|
| **RLHF** | 训练 Reward Model → PPO 优化 | 效果最好 | 需要独立的 RM + 训练不稳定 |
| **DPO** | 直接用偏好数据优化（无需 RM） | 简单高效，单卡可跑 | 效果略低于 RLHF |

```
DPO 的极简版理解:
  数据: (问题, 好的回答, 差的回答) 三元组
  目标: 让模型生成好回答的概率 > 差回答的概率
  不需要: 单独训练 Reward Model（DPO 隐式地做了这件事）
```

## 💼 面试必答

**Q: 什么是知识蒸馏？为什么有效？**
> 知识蒸馏是用大模型（Teacher）的输出概率分布来训练小模型（Student）。有效是因为 Teacher 的 soft probabilities 包含了类别之间的关系（暗知识），比 hard label 提供了更丰富的训练信号。

**Q: LoRA 是怎么工作的？**
> 冻结原始权重 W，只训练两个低秩矩阵 A 和 B（rank 通常 16-64）。推理时 W' = W + AB。这样只需要训练不到 1% 的参数就能达到接近全参数微调的效果，因为模型行为变化通常是低秩的。

**Q: 推理蒸馏和普通蒸馏有什么区别？**
> 普通蒸馏学 Teacher 的输出分布；推理蒸馏学 Teacher 的思考过程（CoT/思维链）。推理蒸馏能让 Student 获得 Teacher 的推理能力，而不只是记忆答案。DeepSeek R1 就是用这种方法让小模型获得大模型级别的推理能力。
