## 核心章节框架

### 1. Prompt 基本原则
> 原则1: 写清晰具体的指令（不是"简短"=好）
> 原则2: 给模型"思考时间"（CoT 的工程化应用）

### 2. 六大技巧
> - 分隔符（```, """, <>）防注入
> - 结构化输出（JSON/XML）
> - Few-shot 示例
> - 角色设定（System Prompt）
> - 任务分解（Chain Prompting）
> - 迭代优化

### 3. 常见陷阱
> - Prompt 太长 → 模型忽略中间内容（Lost in the Middle）
> - 指令矛盾 → 模型随机选一个
> - 要求"不要做X" → 模型反而做了

### 4. Temperature / Top-p 选择指南
> 不同场景该用什么参数。
