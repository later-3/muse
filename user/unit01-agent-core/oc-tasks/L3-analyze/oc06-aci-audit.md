## ACI 六原则回顾 (01a §2.6)

1. **名字清晰** — 工具名一看就知道干什么
2. **描述准确** — description 告诉 LLM 什么时候该调
3. **参数有约束** — enum/required/description 都要有
4. **返回值可预测** — 成功和失败都有格式
5. **错误可恢复** — 报错后 LLM 知道怎么处理
6. **防呆设计 (Poka-yoke)** — 让犯错变得不可能
