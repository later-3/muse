## 🎤 面试卡片（5 道）

### 卡片 1：Tokenization + 成本

**Q: 为什么中文 Agent 比英文 Agent 调用成本更高？技术上怎么解释？**

> **参考答案要点：**
> - LLM 的 Tokenizer（BPE）训练数据以英文为主（~90%+）
> - 英文常见词被充分合并成高效 token（"Hello" = 1 token）
> - 中文合并不充分，每个字 ≈ 2-3 个 token
> - API 按 token 计费 → 同等信息量中文消耗更多 token → 更贵
> - Context Window 也是按 token 计 → 中文可放入的信息更少

> **追问：** 怎么缓解？ → 选择中文优化的模型（Qwen/DeepSeek）；精简 Prompt；工具返回值做摘要压缩

---

### 卡片 2：Agent 定义 + ReAct

**Q: 用一分钟解释什么是 AI Agent，它的核心循环是什么？**

> **参考答案要点：**
> - Agent = LLM + 工具 + 循环
> - 核心循环（ReAct）：Reason(思考) → Action(行动) → Observe(观察) → 重复
> - 和普通 LLM 调用的区别：普通调用一次就结束，Agent 可以多轮循环直到任务完成
> - 三大组件（Weng）：Planning(规划) / Memory(记忆) / Action(工具)

> **追问：** Muse 是一个 Agent 吗？ → 是。engine.mjs 实现循环，memory.mjs 管记忆，MCP 提供工具。

---

### 卡片 3：Function Calling

**Q: LLM 是怎么调用外部工具的？请描述技术流程。**

> **参考答案要点：**
> 1. 开发者定义 Tool Schema（名称 + 描述 + 参数 JSON Schema）
> 2. LLM 收到问题 + 工具列表 → 输出 JSON 格式的 `tool_calls`
> 3. 系统解析 tool_calls → 执行对应函数 → 拿到结果
> 4. 结果作为 `role: tool` 消息回填到对话历史
> 5. LLM 在下一轮看到结果 → 继续推理或给出最终回答
>
> **关键：** LLM 不执行工具！它只生成调用指令。系统负责解析和执行。

> **追问：** 工具描述写得差会怎样？ → LLM 选错工具或传错参数。ACI = 给 LLM 用的 UX。

---

### 卡片 4：BEA 编排 + 设计决策

**Q: 如果让你设计一个 AI 编程助手，你会用 Anthropic BEA 的哪种模式？**

> **参考答案要点：**
> - 核心用 **Orchestrator-Workers**：
>   - Orchestrator 分析需求，决定修改哪些文件
>   - Workers 分别处理各个文件的修改
> - 加 **Evaluator-Optimizer** 层：
>   - 修改完代码 → 运行测试 → 不通过 → 反馈给 Worker 重新修改
> - 遵循 Anthropic 的核心原则："Start simple"
>   - 先实现单文件修改（Prompt Chaining）
>   - 需要多文件时升级到 Orch-Workers
>   - 需要质量保证时加 Evaluator

> **追问：** Prompt Chaining 和 Orchestrator-Workers 的本质区别？ → 步骤固定 vs 步骤动态生成。

---

### 卡片 5：CoT + Reasoning Model

**Q: Chain-of-Thought 为什么能提升 LLM 的推理能力？和 Reasoning Model 有什么区别？**

> **参考答案要点：**
> - **CoT 原理：** 每个中间步骤都经过 Transformer 全部层的计算 → 等效计算量翻 N 倍 → 能解决更复杂的问题
> - **门槛：** 只在 >100B 参数的大模型上有效。小模型加 CoT "想不清楚"反而更差。
> - **CoT vs Reasoning Model 的区别：**
>
> | 维度 | CoT | Reasoning Model |
> |------|-----|-----------------|
> | 实现 | Prompt 技巧（人写） | 训练出来的能力（RL） |
> | 训练 | 不需要额外训练 | 需要 GRPO/PPO 训练 |
> | 思考 | 可见（在输出中） | 通常隐藏（thinking token）|
> | 效果 | 中等提升 | 大幅提升 |

> **追问：** Inference-Time Scaling 是什么？ → 训练阶段 Scaling Law 遇到瓶颈后的新维度 — 在推理阶段花更多计算换更好答案。
