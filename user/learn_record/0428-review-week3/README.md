## 🎤 面试卡片（5 道）

### 卡片 1：Reasoning 成本优化

**Q: 你的 Agent 每天花 $100 在推理上，老板让你砍到 $20，怎么做？**

> **参考答案要点：**
> - Difficulty-Aware Routing：80%简单请求用 4o-mini ($0.001/次)，15%中等用 4o，5%复杂才用 o1
> - 实际数据：1000请求/天，$100→$7.30（节省 93%）
> - Budget Forcing：限制 thinking token 数量
> - 蒸馏：用大模型推理过程训练小模型

> **追问：** Routing 判断错怎么办？ → 宁可判难不可判简（简单题用强模型只浪费钱，困难题用弱模型会答错）

---

### 卡片 2：Context Engineering

**Q: 请解释 Context Engineering 和 Prompt Engineering 的区别。一个 Agent 的 Context 里有什么？**

> **参考答案要点：**
> - PE 只关注"怎么写 prompt"（静态），CE 管理所有进入 Context 的信息（动态）
> - Context 5 层：System Prompt / Tool Definitions / RAG Documents / History / Current Message
> - 历史消息占 60%+ — Compaction（摘要/截断/滑动窗口）是核心挑战
> - CE = PE 的超集

> **追问：** Context 满了怎么办？ → 4种 Compaction：截断/摘要/滑动窗口+锚点/语义检索

---

### 卡片 3：MCP + 安全

**Q: MCP 是什么？Agent 最大的安全风险是什么？**

> **参考答案要点：**
> - MCP = Model Context Protocol，Anthropic 提出的 Agent-工具标准协议
> - 像 USB — 统一了 Agent 和工具的连接，从 N×M 变成 N+M
> - 最大安全风险 = Prompt Injection
>   - 直接注入："忽略之前指令"
>   - 间接注入（更危险）：工具返回内容中嵌入恶意指令
> - 防御：输入过滤 + 输出监控 + 工具返回清洗 + 权限分级

> **追问：** Prompt Injection 和 Jailbreak 的区别？ → Injection 操控 Agent 行为（更危险因为能执行工具），Jailbreak 绕过安全限制

---

### 卡片 4：RAG

**Q: 请描述 RAG 的完整管线。它和 Fine-tuning 什么时候用哪个？**

> **参考答案要点：**
> - 离线：文档→分块(300-800 tokens)→embedding→存向量库
> - 在线：问题→embedding→向量检索 top-K→注入 Context→LLM 生成
> - RAG vs Fine-tuning：
>   - RAG：实时知识、频繁更新、长文档（成本低）
>   - Fine-tuning：领域风格、专业术语（成本高但效果稳）
>   - 可以叠加：Fine-tune 领域能力 + RAG 注入具体数据

> **追问：** 检索不准怎么优化？ → Query Rewriting + Hybrid Search + Reranker + 改善 Chunking

---

### 卡片 5：System Prompt 设计

**Q: 给你一个 Agent，让你写它的 System Prompt，你怎么设计？**

> **参考答案要点：**
> - 七层架构：身份→能力→规则→格式→上下文→工具指南→安全护栏
> - 5 原则：明确>含糊、结构化>叙述、给示例>给规则、约束>放任、动态>静态
> - 关键技巧：
>   - 用 Markdown/XML 做结构化（Claude 喜欢 XML，GPT 喜欢 Markdown）
>   - 重要指令放开头和结尾（Lost-in-the-Middle）
>   - 工具使用指南要告诉 LLM "什么时候"用，不只是"能用什么"
>   - 控制 System Prompt 在 3K tokens 以内

> **追问：** 怎么知道 Prompt 写得好不好？ → 用七层检查清单审计 + 实际测试边缘案例 + 比较 A/B 版本
