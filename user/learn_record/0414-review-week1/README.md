## 🎤 面试卡片（5 道）

### 卡片 1：反向传播

**Q: 用大白话解释反向传播是什么？链式法则在其中扮演什么角色？**

> **参考答案要点：**
> - 前向传播算出预测值 → 和正确答案比较得到 Loss
> - 反向传播 = 从 Loss 出发，**倒着**算每个参数对 Loss 的贡献（偏导数）
> - 链式法则：复合函数求导 → ∂L/∂w = ∂L/∂y × ∂y/∂w
> - 梯度下降：w_new = w_old - lr × ∂L/∂w

> **追问：** 梯度消失和梯度爆炸是什么？ → 链太长导致梯度连乘趋近于 0 或无穷大。解法：残差连接、LayerNorm、梯度裁剪。

---

### 卡片 2：Self-Attention

**Q: 请从底层解释 Self-Attention 的 Q/K/V 机制。为什么不直接用原始向量做注意力？**

> **参考答案要点：**
> - Q = "我在找什么"，K = "我能提供什么"，V = "我的内容"
> - α_{i,j} = softmax(q_i · k_j / √d_k) → 注意力权重
> - b_i = Σ α_{i,j} × v_j → 加权求和
> - 为什么要 QKV 而非原向量？→ 三个独立投影让模型能学到不同粒度的"查询-匹配-提取"关系。如果直接用原向量，搜索和内容纠缠在一起，表达力不够。
> - 为什么要 √d_k？→ 内积随维度增大而增大，softmax 会变成 one-hot，梯度消失。除以 √d_k 稳定分布。

> **追问：** Multi-Head 解决什么问题？ → 一个 head 只能学一种注意力模式（如"看前面的词"或"看语义相近的词"）。多个 head 各看不同面向，拼接后更丰富。

---

### 卡片 3：Transformer Block 完整结构

**Q: 请画出一个 Transformer Block 的完整结构并解释每个组件的作用。**

> **参考答案要点：**
> ```
> Input → [Multi-Head Self-Attention] → Add & LayerNorm → [FFN] → Add & LayerNorm → Output
>              ↑ Residual Connection ↑                      ↑ Residual Connection ↑
> ```
> - **MHA**：捕捉 token 之间的关系（"谁和谁相关"）
> - **FFN**：两层全连接，升维→非线性→降维。被认为是**知识存储**的主要位置
> - **Residual**：跳过连接 → 梯度直通 → 防止梯度消失 → 能堆很多层
> - **LayerNorm**：数值稳定化，防止内部激活值偏移
> - **Causal Mask**（Decoder 专用）：只能看到之前的 token，不能偷看未来

> **追问：** Encoder 和 Decoder 的 Attention 有什么区别？ → Encoder 双向（全看），Decoder 单向（mask 未来）。Cross-Attention: Q 来自 Decoder, K/V 来自 Encoder。

### 卡片 4：训练管线（Pretrain → SFT → RLHF）

**Q: 一个 ChatGPT 级别的 LLM 从"什么都不会"到"能聊天"经历了哪几个训练阶段？分别解决什么问题？**

> **参考答案要点：**
>
> | 阶段 | 输入 | 解决什么 | 类比 |
> |------|------|---------|------|
> | Pretrain | 海量文本（TB级） | 学会语言规律 | 学龄前读遍所有书 |
> | SFT | 人工标注的指令-回答对 | 学会听指令的格式 | 上学学"怎么回答问题" |
> | RLHF (RM+PPO) | 人类偏好排序 | 学会说得好 | 出社会受评价 |
>
> - 关键陷阱：SFT 的目标函数和 Pretrain **一样**（都是 Next Token Prediction + Cross-Entropy），但数据不同。SFT 改变的是**行为模式**而非知识。
> - DPO 是 RLHF 的简化替代：不训 RM，直接从偏好数据优化。

> **追问：** RLHF 中的 KL 惩罚是干什么的？ → 防止模型为了讨好 RM 而说出不自然的话（Reward Hacking）。约束 RLHF 后的模型不要离 SFT 模型太远。

### 卡片 5：灾难性遗忘与 Model Merging

**Q: 如果你 fine-tune 一个英文 LLM 学中文，发现英文能力下降了。原因是什么？怎么解决？**

> **参考答案要点：**
> - **原因**：灾难性遗忘 — 参数更新方向冲突 + 内部特征漂移 + 旧知识连结被覆盖
> - **解法**（4 种）：
>   1. **混合训练**：中英数据按比例混合（简单但需要旧数据）
>   2. **LoRA 隔离**：给中文任务单独的低秩参数，原模型冻结
>   3. **Model Merging**：中文和英文各自独立 fine-tune，用 Task Vector 合并 → θ + α·τ_中文 + β·τ_英文
>   4. **知识蒸馏**：用旧模型的输出当软标签，指导新模型训练

> **追问：** Model Merging 的数学基础是什么？ → Task Vector τ = θ_finetune − θ_base。多个 Task Vector 在参数空间中大致正交时，可以直接相加。
