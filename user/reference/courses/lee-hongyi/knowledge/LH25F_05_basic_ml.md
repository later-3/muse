## 核心教学内容提炼

### 1. 机器学习 = 从数据中找函式

[Fact] 李宏毅把机器学习的本质用一句话概括：

> **"根据资料找出函式 f"**

整堂课围绕一个真实例子展开：**用投影片页数预测课程时长**。他自己收集了过去 ML2021、GenAI2024 的每堂课数据（页数 vs 时长），亲自标注（他笑称 ChatGPT Agent 给的答案都是错的，最后还是人工做的）。

### 2. 找函式的「步骤 3+1」框架

[Fact] 核心框架：

```
步骤一：我要找什么？ → 定义 Loss（给一个 f，我怎么知道它好不好）
步骤二：我有哪些选择？→ 定义模型（框定候选函式的集合）
步骤三：选一个最好的 → Optimization（从集合中挑 Loss 最低的）
+1：  验证（Validation）→ 模拟考 vs 真正的大考
```

### 3. Gradient Descent 的直觉讲解

[Fact] 李宏毅的通俗解释：

> 在 Loss 曲线上，你站在某个点，怎么知道往左走还是往右走？
> - **不是左右各踏一小步去试**（那太麻烦了）
> - 而是**计算切线斜率**（∂L/∂w₁）
> - 斜率是负的 → 右边低 → 往右走
> - 斜率是正的 → 左边低 → 往左走
> - 斜率是 0 → 无路可走 → 停下来

他特别强调：**就算你微积分全忘了也没关系，PyTorch/TensorFlow 都能自动算切线斜率。**

### 4. 「概念很简单，做起来不容易」— Learning Rate 的坑

[Fact] 李宏毅在 Colab 里现场演示踩坑：

| 设置 | 现象 | 他的反应 |
|------|------|---------|
| η=0.01 | w₁ 和 b 全部大爆炸，loss 爆炸 | "不行！" |
| η=0.001, 100 epochs | loss=263（穷举最优是 240） | "离 240 还有一点距离" |
| η=0.001, 1000 epochs | "怎么还在 261？只走了一点点" | "这样不行" |
| η=0.001, 10000 epochs | loss 从 261 到 249 | "设大一点又会飞起来，设小了走太慢" |

> **他的结论：** "Learning rate 设小的时候走得非常慢，但是设大一点又会飞起来。这招不是一个非常好的招数，有没有更好的招数？这个我们留着下周再讲。"

### 5. 从线性到深度学习——Piecewise Linear 逼近

[Fact] 这是李宏毅最精彩的推导路径之一：

```
问题：y = w₁x₁ + b 只能画直线，但真实数据可能是曲线！
  ↓ 怎么办？
思路：在曲线上点一些点，用折线（piecewise linear curve）去逼近
  ↓ 点不够多就不像？
答案：点够多就能无限接近任何曲线！
  ↓ 数学上怎么表示？
每一段折线 = 一个常数 + 一堆"山坡形函数"的叠加
  ↓ "山坡形函数"长什么样？
Sigmoid（S 形曲线）或 ReLU（max(0, x)，折角形）
  ↓ 每个 Sigmoid/ReLU = 一个 Neuron
多个 Neuron 排成一排 = 一个 Layer
多个 Layer 堆起来 = Deep Learning（深度学习）
```

> **关键类比（李宏毅原话）：** "如果说今天 x 跟 y 之间真正的关系是这样一条曲线，在线性函式范围里面，我们不管怎么找怎么找怎么找，都找不出这么一条线。所以看来，这一个函式选择的范围有点太狭隘了。"

### 6. Overfitting — 李宏毅的驾训班类比

[Fact] 这是他最经典的类比之一：

> "有没有考过汽车驾照？在驾训班做训练 = 机器学习的训练过程。考驾照 = 验证过程。正常学车应该看着路况来开。但是有些人会背轨道——驾训班有一些特定的弯道，你可以记住'到第二个锥形筒的时候方向盘打两圈'。这样在驾训班得到满分，但到了真正的道路上就完全不会开了。**这就是 Overfitting！**"

[Fact] 他还举了 Lazy Function 的例子：
- 训练数据上看到的输入 → 输出正确答案
- 没看过的输入 → 一律输出 0
- 这个函数在训练集 loss=0（完美！），但验证集上大爆炸

> "函数的选择范围是无限大的话，你很有机会找到一个函数，它在训练资料上面 loss 为 0，但在验证资料上面一点用处都没有。"

### 7. Validation vs Testing — 鬼灭之刃类比

[Fact] 李宏毅用鬼灭之刃讲为什么不能反复在测试集上调参：

> "就好像在鬼灭之刃里面一样——鬼是杀不死的，人类只有一次机会。人被砍一刀就会死掉，但鬼只要不砍他的头他就可以无限再生。你在 benchmark 上测 2000 天测个十年，终究有一天你会 overfit 在测试资料上。"

因此竞赛设计：
- **Public Set**：可以看分数，但上传次数有限 → "有限次数的验证集"
- **Private Set**：只有一次机会，见真章

### 8. Batch 训练的实际意涵

[Fact] 李宏毅解释 Batch 和 SGD：

```
Full Batch (batch_size = 全部数据)：每看完所有数据才更新一次参数 → 稳但慢
SGD (batch_size = 1)：每看一笔数据就更新一次 → 快但不稳
Mini-batch (如 batch_size = 5)：折中方案
```

每个 epoch 开始时要做 Random Shuffle，确保每次同一个 batch 里的数据不同。

### 9. 实战 Demo — 预测课程时长

[Fact] 他用收集的真实数据现场跑了一个 demo：
- 特征：投影片页数（84 页）、总字数（3388 个）
- 预测这堂课的时长
- **结果：预测值正好是 107 分钟（真实值也是 107！）**
- 他说："这个我真的没有 set 好，这个只是纯粹运气好而已。"
