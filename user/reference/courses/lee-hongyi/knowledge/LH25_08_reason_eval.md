## 核心教學內容提煉

### 1. 評估 Reasoning 模型的特殊困難

[Fact] Reasoning（深度思考）模型的評估比普通 LLM 更難，因為：

- 要看的不只是最終答案，還有**思考過程**是否合理
- 模型可能"答案對但推理錯"（靠猜對的），也可能"推理對但答案錯"（計算失誤）
- 深度思考過程的長度和風格因模型而異，難以標準化比較

### 2. Benchmark 的陷阱 — 數據洩漏

[Fact] 用 Benchmark 評估模型的最大風險之一是**數據洩漏**：

> "Benchmark 的題目可能早就出現在模型的訓練數據中。模型看起來表現很好，但其實是因為它在訓練的時候已經看過答案了。"

李宏毅建議要特別注意 benchmark 的時間線 — 如果 benchmark 在模型訓練數據截止日期之前就公開了，結果就不可信。

### 3. 怎麼計算分數

[Fact] 評估方法的三個層次：

- **與標準答案比對**：最簡單但最粗暴，適合有明確答案的數學/編程題
- **人類評估**：最可靠但成本高、不可複現
- **用 LLM 評估 LLM**：用一個強模型當"考官"打分，但也有偏見（傾向喜歡和自己風格相似的回答）

### 4. 評估的不同面向

[Fact] 不止要評估"聰不聰明"，還要評估：

- **安全性**：會不會生成有害內容
- **抗攻擊能力**：能不能擋住 prompt injection（直接攻擊 vs 間接攻擊）
- **偏見**：改變性別/種族等屬性後，回答是否顯著不同

> "Indirect prompt injection attack 就是攻擊的指令藏在環境中，AI Agent 在跟環境互動的時候會不小心讀到，然後就做出錯誤的行為。"

### 5. 評估 Matrix 不能完全相信

[Fact] 李宏毅反覆提醒：

> "不能夠完全相信 Evaluation 的 Matrix。"

原因：benchmark saturation（天花板效應）、數據洩漏、刷分（針對 benchmark 優化而非真實能力提升）。
