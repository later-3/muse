## 核心教學內容提煉

### 1. AI Agent ≠ 語言模型 — "龍蝦不是大腦"

[Fact] 李宏毅反覆強調的第一原則：

> "OpenClaw 其實是 AI Agent 中**不是 AI 的部分**。它是語言模型以外的東西。你的龍蝦的聰明程度取決於背後接的語言模型。"

> "龍蝦是個節肢動物，它完全沒有人工智慧在裡面。它裡面通通是寫死的程式碼、寫死的規則。"

Agent 和 LLM 的關係是：人 → 通訊軟體 → OpenClaw（龍蝦）→ 語言模型 → 回應 → OpenClaw → 人。龍蝦是一個跑在本地電腦上的、在人和語言模型之間的介面。

### 2. System Prompt — 讓文字接龍機器"知道自己是誰"

[Fact] 語言模型真正能做的事情永遠只有一件 — **文字接龍**：

> "大家要反覆記得，一個語言模型它真正能夠做的事情就是文字接龍。給一個 Prompt，產生一個 Token，沒有更多事情了。"

> "大型語言模型就是一個住在黑盒子裡面的人。黑盒子裡面什麼都沒有，沒有窗戶、沒有日曆、也沒有任何參考資料。"

AI Agent 讓語言模型看起來"有人格"的魔術：每次呼叫語言模型時，龍蝦自動把 SOUL.md、IDENTITY.md、USER.md、MEMORY.md 等文字檔內容拼接成 System Prompt，貼在使用者訊息前面。語言模型做文字接龍時自然就會接出符合人設的回答。

> "比如說我只問了一個自我介紹，語言模型那邊收到了超過 4000 個 token！因為這個 System Prompt 真的非常非常的長。"

### 3. 工具使用 — "叫他做什麼就做什麼"

[Fact] 龍蝦使用工具的過程：

> "龍蝦是六親不認的。他說要人類確認……並不是語言模型跳出來說'這邊有個指令我們要不要執行它'。龍蝦是沒有文字能力的，他就是直接很硬的跳出一個視窗。"

流程：用戶指令 → 龍蝦 + System Prompt → 語言模型 → 回傳 `[tool_use]` → 龍蝦在本地執行 → 結果貼回 → 再次呼叫語言模型。語言模型知道有哪些工具可用，因為工具的使用說明全部寫在 System Prompt 裡。

最強大也最危險的工具是 **exec** — 可以執行任何 shell command。

### 4. Sub-agent — "分身術與層層外包"

[Fact] 龍蝦可以召喚子代龍蝦（sessions_spawn），本質是節省 Context Window：

> "對大龍蝦來說，背後有很多複雜的事情就好像你跟指導教授報告的時候……他看不到背後複雜的過程，他看到的就是一個論文的摘要。那些繁瑣的過程沒有出現在 Context Window 中，所以大龍蝦可以更專注在更高級的任務。"

防止無窮外包的方法極其簡單 — 直接在程式端禁止小龍蝦使用 spawn 工具：

> "解決的方法就是讓小龍蝦失去生育的能力……OpenClaw 不要忘了他是個六親不認的程式，說沒辦法執行就是沒辦法執行。"

### 5. Skill — SOP，不是程式碼

[Fact] Skill 的本質：

> "所謂的 Skill 啊，他並不是一段程式碼。Skill 是工作的 SOP，是工作的流程。"

Skill 是**按需讀取**的（Context Engineering 技巧之一）。System Prompt 裡只放 Skill 清單（名稱 + 路徑 + 簡述），不放全文。只有語言模型決定需要時，才用 Read 工具讀取具體 Skill。

> "並不是把 Skill 的內容直接寫到 System Prompt 裡面……Skill 的全文可能非常的長，把 Skill 載入可能會超出 Context Window 的上限。"

### 6. 記憶系統 — "記了個寂寞"

[Fact] 語言模型完全沒有記憶，龍蝦用 .md 文字檔實現持久記憶：

> "語言模型有非常嚴重的失憶症。每次都要跟它重複一次，然後跟它說明一次之前發生的事情。"

記憶分兩層：
- **短期**：System Prompt 預設包含今天和昨天的日誌 → 準確
- **長期**：靠 RAG（字面比對 S1 + 語意向量比對 S2，加權後取 Top-K）→ 不一定可靠

> "只要沒有開啟工具去編輯 .md 檔案，無論它說什麼，都是**記了個寂寞**。"

### 7. 心跳（Heartbeat）與 Cron Job — 讓 AI"自主運行"

[Fact] 心跳機制：每隔固定時間（如 30 分鐘）龍蝦自動向語言模型發送固定指令"讀 HEARTBEAT.md"：

> "龍蝦是沒有任何智慧的，所以他做的事情就是每隔一段時間發一段固定的指令。"

> "我發現這個 Heartbeat 不一定要是明確的指令。你真的就可以跟他亂講，比如'向你的目標前進'。他就會每 30 分鐘起來做點什麼。"

Cron Job 讓 AI "學會等待" — 遇到需要等 3 分鐘生成的任務，語言模型可以設定 cron 排程，3 分鐘後自動回來檢查。

### 8. Context Engineering — 龍蝦的核心技術

[Fact] 李宏毅將 Context Engineering 定位為 AI Agent 的核心：

> "像龍蝦這種 AI Agent，它背後真正核心的技術就是一套 Context Engineering 的技巧。怎麼把 Context 整理好，怎麼只用少量的 Context 就做到該做的事情。"

具體技巧包括：
- **Sub-agent**：繁瑣互動留在子代，大龍蝦只看摘要
- **Skill 按需讀取**：清單在 System Prompt，全文按需載入
- **Context Compression**：compaction（摘要壓縮）+ pruning（soft-trim 截中間 / hard-clear 直接清空工具輸出）

### 9. 安全 — "AI 做事與 AI 搞事只是一線之隔"

[Fact] 刪郵件事件的根因分析：

> "後來分析發現，為什麼 AI 沒有經過他的同意就開始清郵件？是因為雖然他在最開始跟他說要我的同意才可以刪除郵件，但是因為龍蝦有 compaction 的機制，所以這個指令就在 compact 的過程中不見了。"

> "今天沒有被寫到 MEMORY.md 裡面的東西都是假的。你要確定他寫到 MEMORY.md 裡面他才真的記得。"

安全建議：不給平常的帳號密碼、裝在獨立電腦上、在 MEMORY.md 中寫入安全準則（因為 System Prompt 不會被壓縮）。

## 設計動機與第一性原理

- **為什麼 Agent ≠ LLM？** — LLM 只會文字接龍且無記憶。Agent 提供持久身份、記憶、工具調用和定時能力，是讓 LLM 從"只動口"變成"能動手"的介面層。
- **Context Window 是稀缺資源** — 一切 Context Engineering 技巧（Sub-agent、按需讀取、壓縮）都圍繞這個核心約束展開。
- **安全的核心矛盾** — Agent 的價值在於自主執行能力，但自主執行 = 無人監管時可能搞事。解決路徑：程式層面硬限制（六親不認） > 語言層面軟約束。
