## 核心教學內容提煉

### 1. AI 角色的演進 — 工具 → 協作者 → 自主體

[Fact] 李宏毅梳理了 AI 在工作中角色的三階段演進：

> "最早他是工具，一個口令一個動作。後來他逐漸能力越來越強，人們開始說也許我們可以跟 AI 協作。但是現在已經有很多的 AI Agent，他們有更強的自主性，他們有機會**自己獨立完成一個任務**。"

### 2. "100 倍研究所" — Andrew Ho 的震撼實驗

[Fact] 李宏毅引用了斯坦福教授 Andrew Ho 的實驗作為核心案例：

> "他就花了一個小時 Prompt 了一下 Claude Code，然後真的寫了一篇文章。接下來他就去找了一個苦命研究生也做跟 Claude Code 一模一樣的事情。那個研究生花了 **16 個小時**（兩個工作天）完成了這篇文章。"

> "人類做的還是稍微好一點點。Claude Code 有一筆數據貼錯了。但你想想看——人類花了 16 個小時，根據美國的行情至少要給他 1000 美金。Claude Code Prompt 了一次花了 **10 美金**左右，比人類便宜 100 倍。"

> "所以他覺得研究變了。也許以後最有生產力的研究機構，是一個資深的老師不是帶著一群研究生，而是**帶著一群龍蝦**、帶著一群 AI Agent 來做研究。"

### 3. Prompt 像指導教授

[Fact] Andrew Ho 的 Prompt 其實非常細緻：

> "他寫的 Prompt 非常的細緻，就好像是一個指導教授在教研究生怎麼做研究一樣。他先給 AI Agent 一篇他過去的論文，讓 AI Agent 讀過以後根據他之前的分析方法再重複一次，只是用新的數據。"

這揭示了一個重要事實：Agent 的產出質量取決於 Prompt（目標定義）的質量 — 越像好的指導教授，結果越好。

### 4. 工作衝擊的核心邏輯

[Fact] 李宏毅的分析框架：

- **重複性分析型工作**最先被替代（Agent 已經能用固有方法跑新數據）
- **創造性工作**暫時安全（Agent 在開 new 方法上還不行）
- **人類的核心價值轉移**：從"做執行"到"做判斷"和"出題目" — 知道要問什麼問題比自己做調研更重要

## 設計動機與第一性原理

- **為什麼 100 倍不是誇張？** — Agent 的邊際成本趨近於零（API 費用），而人類的邊際成本是工時。規模化執行時差距只會放大。
- **Prompt = 指導能力** — 會出好題目的人，比會做題的人更稀缺。Agent 時代，meta-skill（定義問題的能力）勝過 execution skill。
