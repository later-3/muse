## 核心教學內容提煉

### 1. 終身學習的動機 — 學習永不停止

[Fact] 李宏毅開篇定義"終身學習"：

> "這些通用模型當它被釋出之後，你可能會希望這些模型仍然可以**持續學習**，根據我們的需求，它仍然可以持續更新參數。就像有一個人出了社會之後，他的學習並不會停止。"

> "當我們講'學習'這兩個字的時候，指的是**有更新參數**我們才稱之為學習。我今天要討論的並不是下了不一樣的 Prompt 讓模型有不一樣的行為。"

### 2. Foundation Model 到 Fine-tuned Model

[Fact] 李宏毅用了清晰的術語區分：

- **Foundation Model**：預訓練完成的通用模型（如 LLaMA、Gemma、DeepSeek）
- **Fine-tuned Model**：為特定任務微調後的模型
- 終身學習的目標：讓 Foundation Model 在不斷接收新任務後，既學會新能力又不忘舊能力

### 3. 災難性遺忘的深入分析

[Fact] 這堂課比 LH25_06 更深入地分析遺忘機制：

- **任務間干擾**：任務 A 需要的參數更新方向和任務 B 正好相反
- **特徵漂移**：fine-tune 改變了模型內部表示，導致舊任務的輸入被錯誤處理
- **知識擦除**：新數據覆蓋了舊數據建立的連結

### 4. 對抗遺忘的系統化方法

[Fact] 高級的抗遺忘策略（超越 LH25_06 的基礎方法）：

**Replay（經驗回放）**
- 保留少量舊數據，新任務訓練時混入舊數據
- 問題：存儲舊數據有隱私和版權問題

**Parameter Isolation（參數隔離）**
- 為每個任務分配獨立的參數子集
- LoRA 等低秩適配器天然適合這個策略

**Model Merging（模型合併）**
- 見 LH25_11 — 各任務獨立 fine-tune 後合併
- 完全避免遺忘，但受制於同一 Foundation Model 的約束

**Knowledge Distillation（知識蒸餾）**
- 用舊模型的輸出做"軟標籤"來指導新模型的訓練
- 新模型在學新任務的同時模仿舊模型的行為

### 5. Post-training 時代的新挑戰

[Fact] 在大模型時代，終身學習面臨新的研究方向：

- **持續對齊**：模型不斷學習新任務時，如何保持 RLHF 的 alignment 不退化？
- **能力堆疊**：每學一個新技能都應該是在已有能力之上的累積，而非替換
- **遺忘檢測**：如何在不運行所有舊 benchmark 的情況下快速檢測遺忘發生？
