## 核心教學內容提煉

### 1. 評估 GenAI 比傳統 ML 難得多

[Fact] 傳統分類問題有明確的正確答案（accuracy），但生成式 AI 的輸出是開放式的 — 同一個問題可以有無窮多個合理回答。

> "生成式 AI 的評估比傳統 ML 難得多，因為你沒有一個唯一的標準答案。"

### 2. 自動評估指標的局限

[Fact] 李宏毅詳細拆解了常見自動指標的問題：

- **BLEU / ROUGE**：基於 n-gram 匹配，只看字面重疊。語意相同但用詞不同的回答會得低分。
- **Perplexity**：衡量模型對語言的"困惑度"。低困惑度 ≠ 好回答（可能只是背書式複述）。
- **BERTScore / Embedding 相似度**：比字面匹配好，但仍有盲區。

> "你可以計算跟標準答案的相近程度，你可以用人類評估，你可以嘗試用 LLM 來取代人類的評估。"

### 3. LLM-as-Judge 的偏見

[Fact] 用 LLM 當考官打分（LLM-as-Judge）是目前最流行的自動評估方式，但有系統性偏見：

- **自我偏好**：GPT 傾向給 GPT 生成的回答打高分
- **位置偏見**：放在前面的回答容易被打高分
- **長度偏見**：更長的回答被認為更好（但不一定更準確）

### 4. Benchmark 大量失效

[Fact] 李宏毅舉了多個 Benchmark 失效的案例：

- **數據洩漏**：Benchmark 出現在訓練集中，分數虛高
- **Benchmark Saturation**：所有模型都接近滿分，無法區分優劣
- **刷分**：針對 benchmark 做特定優化，分數上去了但真實能力沒提升
- **POMP 敏感**：同一個問題不同的 prompt 格式可能導致分數差異巨大

> "要注意你輸入的 Prompt。要注意資料洩漏的問題。也需要注意惡意攻擊和模型偏見的問題。"

### 5. 人類評估的金標準和代價

[Fact] 人類評估仍然是金標準，但：

- 成本極高（需要付酬金）
- 不可複現（不同人標準不同）
- 有主觀性（評估者自身的偏見）
- 評估者也會疲勞（連續評幾百個後質量下降）

### 6. 安全性評估

[Fact] 除了能力評估，還要評估安全性：

- **有害內容生成**：模型是否會輸出危險/冒犯性內容
- **Prompt Injection 抵抗**：直接攻擊（direct）vs 間接攻擊（indirect — 指令藏在環境中）
- **偏見檢測**：改變輸入中的性別/種族/年齡等屬性，觀察回答變化

## 設計動機與第一性原理

- **評估為什麼是最難的問題？** — 如果你不能正確評估模型，你就無法知道模型在進步還是在退步。一切優化都建立在可靠評估之上。
- **沒有完美的評估方式** — 自動指標有盲區，人類評估太貴，LLM-as-Judge 有偏見。實踐中要多種方式交叉驗證。
- **124 分鐘的課 = 評估真的很重要** — 這是 25F 系列最長的課之一，說明李宏毅認為評估能力是 GenAI 從業者最容易被忽視但最關鍵的技能。
