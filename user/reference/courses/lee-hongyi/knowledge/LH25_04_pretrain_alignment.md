## 核心教学内容提炼

### 1. 三阶段范式：Pretrain → SFT → RLHF

[Fact] 李宏毅开篇即给出全景：

> "今天训练语言模型基本上分成三个阶段：
> 1. **Pretrain** — 透过大量网路爬到的资料，让机器有基本的文字接龙能力
> 2. **SFT (Supervised Fine-Tuning)** — 告诉模型看到某个输入时什么样的输出才是对的
> 3. **RLHF** — 由使用者告诉机器什么样的回答比较好、什么比较不好"

> "第二和第三阶段需要人类参与，我们把这两个阶段叫做 **Alignment**。"

### 2. Alignment 只是「画龙点睛」

[Fact] 这是本讲最核心的洞察：

> "**Alignment 只是画龙点睛，而 Pretrain 的阶段看过大量各式各样的资料才是成功的关键。**"

> "Alignment 往往只是**强化**语言模型本来就知道、本来就会的能力。它难以让模型学会新的技能、给它新的知识。"

### 3. SFT 只改变极少数 Token 的行为

[Fact] 关于 Aligned 前后模型的差异：

> "研究发现 shift token 的比例非常非常的少——也就是说模型在 Aligned 前后的行为其实没有那么大的变化。"

> "你可能会问，为什么答案看起来差那么多但行为没有很大的变化？因为模型做文字接龙的时候就是**一步错步步错**——中间有一个地方错了，接下来接的东西就会非常的不一样。"

### 4. Pretrain 数据品质 > 数量

[Fact] 关于数据清理的重要性：

> "如果在同样 Token 数量的情况下，拥有改写过的 Pretrain 资料来训练模型是比较有效的。如果要达到同样的正确率，没有改写过的资料你需要花另外**三倍的数据量**才有办法得到差不多的能力。"

> "今天你如果要 Pretrain 模型的话，直接从网路上爬到的资料往往没有办法直接用，你需要经过多个不同的步骤进行清理后才能拿到真正可以 Pretrain 模型的资料。"

### 5. Alignment 的极限

[Fact] 核心结论：

> "Alignment 或 Fine-tune 很难教会语言模型新东西。怎么透过微调教语言模型新的东西？怎么做有效的微调？我们就下回分解。"
