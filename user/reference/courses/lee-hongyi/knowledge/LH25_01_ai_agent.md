## 核心教學內容提煉

### 1. AI Agent 的定義 — "人類只給目標"

[Fact] 李宏毅開場就做了一個關鍵區分：

> "今天我們使用 AI 的方式通常是人類給一個明確的指令，一個口令一個動作。AI Agent 的意思是說，人類不提供明確的行為或步驟的指示，人類只給 AI **目標**，至於怎麼達成目標呢，AI 要自己想辦法。"

Agent 需要多步驟與環境互動，並能靈活根據現況調整計畫。

### 2. 從 RL Agent 到 LLM Agent — 為什麼爆紅

[Fact] 以 AlphaGo 為例解釋 Agent 本質（目標→觀察→行動→循環），再指出 RL 的局限：

> "透過 RL 演算法的局限是，你需要為**每一個任務**都用 RL 的演算法訓練一個模型。在經過了大量的訓練以後，它可以下圍棋，但並不代表它可以下其他的棋類。"

> "今天 AI Agent 再次爆紅，並不是真的有了什麼跟 AI Agent 本身相關的新的技術，而是在 LLM 變強之後，人們開始想——我們能不能直接用 LLM 來實現人類擁有一個 AI Agent 的渴望？"

### 3. Agent = 記憶 + 行動（工具）+ 規劃

[Fact] 李宏毅把 Agent 內部拆成三大模組：

**記憶模組**：短期記憶（當前對話上下文）+ 長期記憶（外部存儲，RAG 檢索）。還有第三個可選模組 — **Reflection（反思）**：

> "這個模組的工作是對記憶中的資訊做更好的、更 high level 的、可能是抽象的重新整理。你可以把記憶裡面的內容經過 reflection 模組重新反思之後，得到新的想法。"

**行動模組（工具使用）**：語言模型輸出文字形式的 action proposal，系統把它翻譯成真實的 API 調用。

> "產生 Action 的方式可能就是用一段文字。使用工具就是調用函式，所以使用工具又叫做 **function call**。"

**規劃模組**：決定先做什麼、後做什麼。

### 4. ReAct — 思考與行動交織

[Fact] 介紹了 ReAct（Reasoning + Acting）範式，是 Agent 最常見的執行模式：

模型交替做兩件事：
1. **Thought**（推理）：分析當前狀況，決定下一步
2. **Action**（行動）：執行工具或操作
3. **Observation**（觀察）：接收結果
4. 循環直到完成

### 5. 多 Agent 協作模式

[Fact] 李宏毅介紹了多個 Agent 分工協作的架構：

- **垂直分工**：一個 Manager Agent 分配任務給多個 Worker Agents，類似 OpenClaw 的 Sub-agent/Spawn 機制
- **水平協作**：多個 Agent 扮演不同角色辯論或投票
- **自我對弈**：Agent 自己扮演對手做測試

### 6. Agent 的安全與可控性

[Fact] 放權給 Agent 帶來的核心矛盾：

> "通常你期待 AI Agent 要解決的目標、要達成的目標是需要通過多個步驟跟環境做很複雜的互動才能夠完成。而環境會有一些不可預測的地方。"

一旦 Agent 能自主執行，就必須同時解決可控性（不做過頭）和安全性（不做錯事）。

## 設計動機與第一性原理

- **為什麼 LLM Agent > RL Agent？** — LLM 是通用的，一個模型可以處理多種任務，不需要為每個任務單獨訓練
- **Agent 的三大支柱** — 記憶（知道過去）、工具（改變世界）、規劃（決定順序）
- **與 LH26_01 的關係** — 本講是理論框架，LH26_01 是 OpenClaw 的工程實踐
