# 学习助手 V1-V3 路线图

> **原则：** V0 先让 AI 帮你快速做出来；V1 以后尽量由你自己边学边升级。  
> **目标：** 让这个项目成为你的长期练手场。

---

## 一、版本定位

| 版本 | 谁主做 | 目标 |
|------|--------|------|
| **V0** | 其他 AI | 尽快做一个你能用的版本 |
| **V1** | 你 | 把它升级成更像样的学习助手 |
| **V2** | 你 | 加入截图问答 |
| **V3** | 你 | 加入屏幕/区域理解 |

---

## 二、V1：你接手后先做什么

### 推荐升级项

1. 把“整文件加载”升级成“分段检索”。
2. 优化 system prompt，让回答更适合：
   学习
   面试
   笔记总结
3. 增加“解释模式 / 面试模式 / 笔记模式”。
4. 增加最近对话历史持久化。
5. 增加引用来源显示：
   回答用了哪个本地文件、哪一节。

### 对应你会练到的能力

1. Prompt Engineering
2. 简单 RAG
3. Tool/Context 设计
4. 前后端联调

---

## 三、V2：截图问答

### 目标

1. 你上传一张截图。
2. 助手能识别图中内容并回答。
3. 和本地知识库联合回答。

### 推荐范围

1. 先支持手动上传图片。
2. 再支持浏览器内框选后上传裁剪图。
3. 不急着做持续屏幕监听。

### 你会练到

1. 多模态输入
2. OCR / Vision API
3. 图文联合上下文设计

---

## 四、V3：屏幕理解助手

### 目标

1. 允许共享屏幕或手动抓取当前屏幕。
2. 你框选一个区域并提问：
   “这里为什么报错？”
   “这段图表是什么意思？”
3. 助手结合视觉信息和本地知识回答。

### 推荐路线

1. `getDisplayMedia` 屏幕共享
2. 单帧截图
3. 前端框选区域
4. 裁剪图片发送到后端
5. OCR / Vision 模型理解

### 你会练到

1. 浏览器媒体 API
2. 屏幕采集
3. OCR / Vision
4. 多模态 agent 交互

---

## 五、它和 Muse 的关系

1. 这是你的练手项目。
2. 但里面做出来的能力以后都能回灌 Muse：
   语音入口
   知识解释
   截图问答
   屏幕理解

> 所以它不是一次性 demo，而是未来 Muse 多模态入口的试验田。

---

## 六、最重要的执行原则

1. **V0 先用起来，不追求完美。**
2. **V1-V3 每次只做一个明确升级点。**
3. **每学完一个知识点，就在这里落一次实践。**
4. **不要让这个支线拖住 S2b 主线。**
