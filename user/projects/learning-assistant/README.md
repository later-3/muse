# 学习助手项目

> **定位：** 这是 `Muse Self-Development MVP` 主线之外的支线项目。  
> **目标：** 先做一个你能立刻用起来的 `V0 学习助手`，后续再由你边学边迭代成语音/视觉/屏幕多模态助手。

---

## 一、为什么做它

1. 它先服务你自己，立刻提升学习效率。
2. 它是一个小而完整的 Agent 应用练手项目。
3. 它未来可以回灌 Muse：
   语音交互
   知识检索
   笔记沉淀
   视觉输入

---

## 二、产品定位

1. **产品名：** 学习助手（Learning Assistant）
2. **V0 入口：** 语音对话网页
3. **V1-V3 方向：**
   截图问答
   框选区域提问
   屏幕理解辅助

> 结论：它不是“只会说话的语音助手”，而是“以语音为第一入口的学习助手”。

---

## 三、文件说明

| 文件 | 用途 |
|------|------|
| `voice-chat-design.md` | 当前产品/交互设计草案 |
| `open-source-survey.md` | 开源方案调研与选型结论 |
| `v0-ai-build-spec.md` | 给其他 AI 的 V0 开发任务清单 |
| `v1-v3-roadmap.md` | 你后续自己迭代的路线图 |

---

## 四、推荐执行方式

1. **现在：** 让其他 AI 直接按 `v0-ai-build-spec.md` 开发一个可用版本。
2. **你自己：** 用起来，边学边记录不满意点。
3. **后续：** 每学完一个主题，就在这个项目里亲手做一个升级项。

---

## 五、和主线的关系

1. 它**不替代** `Muse Self-Development MVP`。
2. 它是 `sidecar / 支线项目`。
3. 它可以沉淀成你面试时的第二个项目证明：
   “我不只做了 Muse，还做了一个帮助自己学习 Agent 的真实应用。”
