# Unit / OC 实战文档规范

> **适用范围：** `user/unitXX-*/README.md` + `oc-tasks/*.mjs` + `oc-tasks/*.md`
> **核心原则：** 每个 unit 是一个完整的学习单元，结构清晰、可导航、可自检。

---

## 一、Unit README 必须章节（按顺序）

### 0. 标题 + 定位
```markdown
# Unit XX: [主题名]
> **对应 Sprint X Day X-X** · [核心内容关键词]
```

### 1. 📚 前置基础速查表
> **必须有！** 碰到不懂的概念，知道去哪找。

```markdown
| 看到什么不懂 | 去哪里 |
|------------|--------|
| [具体问题] | `foundations/FX` §Y ([具体内容]) |
```

### 2. 学习目标
> **用问题形式！** 读完本 unit 能回答什么。3-5 个核心问题。

### 3. 📖 学习文档 → `study/`
> 表格形式。每行：文件名 + 一句话内容 + 状态标记。

### 4. 🎯 你的任务
> Later 本人需要完成的沉淀任务。Checkbox 格式。

### 5. 🤖 AI 并行任务 → `experiments/`
> 表格形式。类型(🧪/🔧) + 文件名 + 结果。

### 6. 🔧 OC 实战任务 → `oc-tasks/`
> **三大类 ABC 分组，每类至少 3 个任务。**

```markdown
### A. 理论实操 — 把核心概念变成代码
| # | 任务 | 对应理论 | 产出 |
|---|------|---------|------|

### B. 课程练习 — 跟优质课程动手
| # | 来源 | 练什么 | 产出 |
|---|------|--------|------|

### C. 项目拆解 — 拆真实项目理解落地
| # | 项目 | 拆什么 | 产出 |
|---|------|--------|------|
```

### 7. 🏗️ 并行项目里程碑
> Muse + 学习助手在本 unit 对应的里程碑。表格形式。

---

## 二、OC 实战代码规范 (`oc-tasks/ocXX-*.mjs`)

### 2.1 文件头（必须）

```javascript
/**
 * ocXX — [一句话说清楚这个任务做什么]
 *
 * 📖 对应 Unit XX: [对应的学习内容]
 * 🎯 实战目标: [具体要实现/验证什么]
 * 📦 类型: A.理论实操 / B.课程练习 / C.项目拆解
 *
 * 📌 核心概念:
 * - [概念1英文(中文)]: [一句话解释]
 *
 * 📚 参考资源:
 * - [课程/项目名]: [具体 URL 或文件路径]
 *
 * 🔧 运行方式:
 *   node user/unitXX-*/oc-tasks/ocXX-*.mjs
 *   node --test user/unitXX-*/oc-tasks/ocXX-*.test.mjs
 *
 * 💡 关键收获: [完成后填写]
 */
```

### 2.2 代码内注释要求

同 `ai-parallel-task.md` 的 §1.2，额外要求：
- **参考来源标注**: 如果代码来自课程/项目，标注原始出处
- **简化说明**: 如果对原代码做了简化，说明省略了什么、为什么可以省
- **可独立运行**: 每个 OC 任务必须能单独跑，不依赖其他 OC 任务

## 三、项目拆解报告规范 (`oc-tasks/ocXX-*.md`)

### 3.1 必须章节

```markdown
# ocXX: [项目名] — [拆解什么]

> **项目**: [名称 + GitHub URL]
> **拆解模块**: [具体模块/文件]
> **对应 Unit**: Unit XX [主题]
> **核心问题**: 这个项目是怎么实现 [理论概念] 的？

---

- 和我们学的理论怎么对应: [一句话]

## 2. 架构拆解
[用图 + 文字拆解核心架构]
[标注：哪部分对应 unit 的哪个理论]

## 3. 关键代码走读
[贴核心代码（标注文件路径 + 行号）]
[用注释解释每一步在干什么]
[标注：这里用了什么设计模式/理论]

## 4. 设计决策分析
| 决策 | 他们怎么做的 | 为什么这么做 | Muse 是否适用 |
|------|-----------|-----------|-------------|

## 5. Mini 复现
[简化版实现，可独立运行]
[标注：省略了什么、核心保留了什么]

## 6. 💼 面试怎么讲
- "这个项目的 [模块] 是怎么实现的？"
- [标准答案，1-2 分钟口述版]

## 7. Muse 启发
- [对 Muse 的哪个模块有什么启发]
- [建议后续怎么借鉴]
```

## 四、自检清单

- [ ] 有前置基础速查表吗？
- [ ] 学习目标是问题形式吗？
- [ ] OC 任务分了 ABC 三大类吗？

- [ ] 课程练习标注了具体来源吗？
- [ ] 项目拆解标注了具体模块吗？
- [ ] 并行项目里程碑和 unit 内容对应吗？

- [ ] 文件头有完整 JSDoc 吗？
- [ ] 标注了对应 unit 和理论吗？
- [ ] 标注了参考资源 URL 吗？
- [ ] 能独立运行吗？
- [ ] 有测试文件吗？

### 项目拆解报告自检
- [ ] 有 30 秒速读吗？
- [ ] 有架构图吗？
- [ ] 关键代码标注了文件路径 + 行号吗？
- [ ] 有 mini 复现代码吗？
- [ ] 有面试口述版吗？
- [ ] 有 Muse 启发吗？
