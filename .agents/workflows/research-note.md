# 研究笔记撰写标准

> **适用范围：** `user/research/` 下所有 `.md` 文件
> **核心目标：** 每一分钟的学习投入都要有最大化的回报 — 同时服务于「积累知识经验」和「打造 Muse」两个核心目标

---

## 一、必须包含的章节（按顺序）

### 1. 元信息头（Meta Header）
```
来源 URL / 论文标题 / 作者
Sprint / Day / 类型（精读 / 走读 / 实验）
一句话总结（中 + 英）
```

### 2. ⭐ 核心概念（Key Concepts）
- 每个概念必须有：**英文原词 + 中文翻译 + 一句话解释**
- 重要概念用 ⭐ 标注
- 按概念依赖关系排序（先学什么后学什么）
- 列出「前置知识」依赖

### 3. 📖 原文精读（Deep Reading）
- **重要段落必须中英双语对照**（英文原文 blockquote + 中文解读）
- 非重要内容一笔带过，不浪费篇幅
- 用 `> [!IMPORTANT]` 标注最关键的洞察

### 4. 🔍 开源项目实战映射（Real-World Projects）
- 每个核心概念 → 至少映射 1 个开源项目的具体实现
- 展示核心代码片段或流程图（伪代码也行）
- 分析：为什么这么设计？效果好在哪？缺点是什么？
- 项目池（持续扩充）：
  - Swarm (OpenAI), LangGraph (LangChain), CrewAI, AutoGen (Microsoft)
  - Claude Code, Codex CLI, Gemini CLI, OpenCode, Clowder AI
  - 字节 Coze, 阿里通义千问 Agent, Dify, Agno
  - 其他行业知名项目

### 5. 🏗️ Muse 深度思考（Muse Implications）
- 不止是「对照表」，要有具体的设计思考：
  - 这个概念对 Muse 的哪个场景（S1/S2/S3）有什么启发？
  - 现有代码哪里需要调整？为什么？
  - 如果要借鉴，Muse 的约束条件是什么？（OpenCode 底座、Telegram、MCP）
  - 记录为未来 Spike/Design 的输入

### 6. 👤 人物与术语（People & Terminology）
- 提到的关键人物：简短介绍（谁、在哪、做了什么、为什么重要）
- 专业术语表：中英对照 + 上下文说明
- 目标：面试时像业内人士一样使用这些术语

### 7. 💼 面试准备（Interview Prep）
- **基础概念题**：面试官可能问的定义类问题 + 标准答案
- **设计题**：「如果让你设计一个 xxx，你会怎么做？」
- **故事题**：基于 Muse 项目的技术故事（STAR 格式）
- **对比题**：xxx 和 yyy 有什么区别？各自适合什么场景？
- **陷阱题**：常见误区，面试时容易答错的点

### 8. ❌ 常见误区（Common Misconceptions）
- 小白容易搞错的概念
- 似是而非的理解

### 9. ✅ 自检题（Self-Test）
- 3-5 道检验理解的问题
- 带参考答案（折叠）

### 10. 📚 延伸阅读（Further Reading）
- 分层：入门 / 进阶 / 深水区
- 标注优先级

---

## 二、格式规范

### 视觉层级
- ⭐ = 最重要，必须掌握
- 🔑 = 关键洞察
- 💡 = Muse 启发
- ⚠️ = 注意陷阱
- 重要内容用 `> [!IMPORTANT]` 或 `> [!TIP]` 高亮
- 非重要内容简短概括，不展开

### 英文对照
- 关键段落：英文原文 blockquote + 中文解读
- 概念首次出现：**English Term（中文翻译）**
- 不需要全文翻译，只对照重要部分

### 代码展示
- 开源项目代码：标注来源仓库和文件路径
- 用伪代码或简化版展示核心逻辑
- 加注释说明设计意图

---

## 三、长度控制

- 单篇笔记不超过 800 行
- 如果内容太多，拆分为：
  - `xxx.md` — 主笔记（概念 + 精读 + Muse 思考）
  - `xxx-projects.md` — 项目实战分析（代码走读）
  - `xxx-interview.md` — 面试准备专篇
- 拆分时主笔记要有链接导航

---

## 四、写完后的检查清单

- [ ] 每个核心概念都有英文原词吗？
- [ ] 重要段落有中英对照吗？
- [ ] 至少映射了 3 个开源项目吗？
- [ ] Muse 思考是否具体到场景和代码层？
- [ ] 面试题覆盖了基础 + 设计 + 故事吗？
- [ ] 有自检题吗？
- [ ] 非重要内容是否足够简短？
