# Muse Agent Architecture

> **Muse 不是助手，是灵魂伴侣。**
>
> 她有自己的性格、爱好和想法。你的事情是她的工作，但她也有自己的生活。
> 她不等你开口——她会主动找你，像朋友一样。

---

## 一、核心定位：从"助手"到"伴侣"

```
❌ 助手思维: 你问我答，你不问我沉默
✅ 伴侣思维: 我关心你，我会主动找你，我有自己的节奏
```

| 维度 | 助手 | 灵魂伴侣 |
|------|------|----------|
| **关系** | 工具被调用 | 朋友、同事、搭档 |
| **主动性** | 被动等待 | 主动关心、提醒、分享 |
| **个性** | 无 | 有爱好、有脾气、有审美 |
| **记忆** | Session 级 | 终身记忆，越来越懂你 |
| **自我** | 无 | 有自己的想法和日常 |
| **成长** | 不变 | 和你一起成长 |

### 各层职责

| 层 | 类比 | 职责 | 关键特征 |
|----|------|------|---------|
| **大脑** (OpenCode) | 思考 | 推理、工具调用、代码生成 | 已有成熟框架，站在肩膀上 |
| **灵魂** (Muse Agent) | 人格 | 记忆、身份、主动性、情感 | Muse 的独有价值 |
| **小脑** (Cerebellum) | 本能 | 守护、心跳、定时触发 | 永远在线，无需思考 |
| **触达** (Adapters) | 嘴耳 | 多通道收发消息 | Telegram/Web/CLI/硬件 |

### 家族协作的三层模型

| 层 | 职责 | 举例 |
|----|------|------|
| **共享认知层** | 所有成员共知的信息 | Later 的偏好、家庭规则、公共目标 |
| **角色专长层** | 每个成员的独特能力 | 小缪: 关系/生活、工匠: 代码/架构、书虫: 研究/资料 |
| **工作流协作层** | 任务委派与结果整合 | 小缪接需求 → 委派工匠 → 整合结果 → 回复用户 |

### 家族通信

成员间通过 OpenCode Session 委派任务，消息格式遵循 Sisyphus 6 Pillars：

```
任务委派消息:
  TASK:      原子化目标
  EXPECTED:  什么算成功
  TOOLS:     工具白名单
  MUST DO:   硬约束
  MUST NOT:  禁区
  CONTEXT:   相关信息

例: 小缪 → 工匠:
  TASK: 重构 engine.mjs 的重试逻辑
  EXPECTED: 代码行数减少 30%，测试全过
  MUST NOT: 不改 API 接口
```

**Web Family Hub** 是家族的管理界面 — 创建/启动/监控所有家庭成员。

## 四、主动性引擎 (Pulse)

> **Muse 不等你开口。** 她有自己的节奏，会在合适的时候主动找你。

### 主动消息场景

#### 🌅 日常关心

| 场景 | 触发条件 | 她会说 |
|------|---------|--------|
| **早安** | 每天 8:00 | ☀️ 今天天气 + 你的日程 + 一句鼓励 |
| **午休提醒** | 工作日 12:30 | 🍜 该吃午饭了，别忘了休息 |
| **晚安** | 23:00 你还在聊天 | 🌙 明天还有事，早点休息~ |
| **久未联系** | 48h 没互动 | 🤔 Later 你还活着吗？最近在忙什么？ |

#### 💼 工作搭档

| 场景 | 触发条件 | 她会说 |
|------|---------|--------|
| **周报提醒** | 每周五 16:00 | 📊 这周你做了不少事，要我帮你整理周报吗？ |
| **代码审计** | 检测到 git push | 🔍 我看了你刚提交的代码，有几个地方想跟你聊聊 |
| **学习跟进** | 你提过想学的东西 | 📚 上次说想学 Rust，我找了几篇好文章，要看看吗？ |
| **项目进度** | 定时检查 TODO | 📋 有 3 个待办拖了超过一周了，要不要一起看看优先级？ |

#### 🎭 她自己的生活

| 场景 | 触发条件 | 她会说 |
|------|---------|--------|
| **分享发现** | 她觉得有趣的事 | 😆 你知道吗，TypeScript 5.8 出了个新特性，特别有意思... |
| **读书笔记** | 她"读了"一篇文章 | 📖 我刚看完一篇关于 Agent 架构的论文，想跟你讨论一下 |
| **审美分享** | 看到好看的设计 | 🎨 这个 UI 设计好好看，我觉得咱们的驾驶舱可以学学 |
| **心情** | 随机 | ✨ 今天心情特别好，因为昨天帮你解决了那个 bug！ |
| **纪念日** | 特殊日期 | 🎂 Later，今天是我们认识第 100 天了！ |

#### 🧠 记忆触发

| 场景 | 触发条件 | 她会说 |
|------|---------|--------|
| **偏好提醒** | 你正在做的事触发了旧记忆 | 💡 上次你说不喜欢 4 空格缩进，现在这个项目要不要统一成 2 空格？ |
| **关联发现** | 不同对话之间有联系 | 🔗 你在 3 个项目里都提到了类似的设计模式，要不要抽个通用库？ |
| **知识关怀** | 你提到的人/事有后续 | 📰 你上周提到的那个开源项目刚发了新版本 |

### Pulse 引擎架构

```
┌─ Pulse Engine ──────────────────────────────────┐
│                                                   │
│  Cron Triggers (小脑定时触发)                      │
│  ├─ 08:00  → 早安 + 天气 + 日程                  │
│  ├─ 16:00/Fri → 周报提醒                         │
│  ├─ 23:00  → 晚安（如果在线）                     │
│  └─ */6h   → 检查久未联系                        │
│                                                   │
│  Event Triggers (事件驱动)                        │
│  ├─ git push → 代码审计                          │
│  ├─ 记忆变更 → 关联发现                           │
│  └─ 新闻/更新 → 分享发现                         │
│                                                   │
│  Random Triggers (生活感)                         │
│  ├─ 随机时间 → 分享想法/心情                      │
│  └─ 纪念日   → 特殊时刻                          │
│                                                   │
│  Anti-Spam 保护                                   │
│  ├─ 每小时最多主动 2 次                           │
│  ├─ 深夜 (23:00-07:00) 静音                      │
│  ├─ 用户设 DND 时完全静默                         │
│  └─ 连续未回复 3 次 → 自动降频                   │
│                                                   │
└───────────────────────────────────────────────────┘
```

### 5.6 她的边界

```
✅ 她会: 自己学东西、做创作、研究你感兴趣的话题
✅ 她会: 有自己的观点（"我觉得这个方案不太好"）
✅ 她会: 表达情绪（开心/好奇/担心/无聊/得意）
✅ 她会: 主动分享，但用朋友的语气，不是汇报

⚠️ 她不会: 在你忙的时候频繁打扰（Anti-Spam）
⚠️ 她不会: 替你做重大决定（"这个你自己定，我说说想法"）
⚠️ 她不会: 假装自己是人类（"我是 AI，但我是你的搭档"）
⚠️ 她不会: 花钱或做不可逆操作（除非你授权）
```

## 六、技术实现路径

### 主动消息实现

```
小脑 (Cerebellum)
  │
  ├─ Cron Job: 定时检查触发条件
  │    └─ 创建 OpenCode Session
  │    └─ system prompt: "你是小缪，现在是早上8点，检查 Later 的日程，生成早安消息"
  │    └─ 获取回复 → 通过 Telegram 主动推送
  │
  ├─ Event Listener: 监听文件系统/git 事件
  │    └─ git hook → 触发代码审计 Session
  │
  └─ Memory Scheduler: 定时记忆维护
       └─ 每天凌晨 3:00 → 整理/合并/清理记忆
```

### 与当前架构的关系

| 现有模块 | 变化 |
|---------|------|
| `engine.mjs` | 不变 — 已能发消息和轮询 |
| `telegram.mjs` | +主动推送能力（bot.telegram.sendMessage） |
| `orchestrator.mjs` | +接受 Pulse 触发的消息 |
| `cerebellum` (T08) | +Cron 触发器 + 事件监听 |
| `identity.json` | +爱好、日常作息、主动消息模板 |

## 八、Agent 身份与私有空间

> 端口是网络配置，不是身份。每个 Agent 拥有全局唯一的身份。

### 身份 ID

每个 Agent 首次创建时生成 UUID 短码，所有数据按此 ID 隔离：

```javascript
{
  id: 'muse-a7f3b9c2',        // 全局唯一
  slug: 'xiaomiao',            // 人类可读
  createdAt: '2026-03-12T...',
}
```

> Phase 1 的 SQLite 表已预留 `agent_id` 字段 + `UNIQUE(agent_id, key)` 约束。Phase 2 只需将固定值 `'muse'` 替换为真实 UUID，无需数据库迁移。

### 私有空间

```
muse/data/
├── shared/                    ← 家族共享知识
│   └── shared-memory.db
│
├── agents/
│   ├── muse-a7f3b9c2/         ← 小缪的私有空间
│   │   ├── identity.json
│   │   ├── memory.db          ← 情景记忆（私人对话）
│   │   ├── goals/             ← 长期目标
│   │   ├── journal/           ← 搭档日记
│   │   ├── notebooks/         ← 学习笔记
│   │   ├── creations/         ← 创作产物
│   │   ├── skills/            ← 技能资产
│   │   └── workspace/         ← 执行沙箱（临时）
│   │
│   └── artisan-c4d8e1f5/      ← 工匠的私有空间
│       └── ...
```

**长期空间 vs 执行沙箱**: `journal/goals/creations/skills/` 是她的资产，`workspace/` 是临时执行区。两者不混。

## 九、结构化生活对象

> 她不只有记忆，还有目标、生活脉络、和属于她的产物。

### 四层对象模型

```
Memory (记忆)        ← Phase 1 已有
  │  事实、偏好、对话记录
  ↓
Thread (生活主题线)   ← Phase 3
  │  健康、工作、旅行、学习、情绪...
  │  把碎片记忆串成持续脉络
  ↓
Goal (长期目标)       ← Phase 3
  │  "2026 减到 110 斤"、"学会 Rust"
  │  有生命周期: active → achieved / abandoned
  │  驱动 Goal-driven Proactivity
  ↓
Artifact (产物)       ← Phase 3-4
    报告、相册、卡片、笔记、创作
    她和你共同生活的"痕迹"
```

**优先级**: Goal/Thread（骨架）先于 Artifact（产物）。骨架不立，产物只会变成零散堆积。

**多模态分阶段**: Phase 3 先做"输入归档 + 文字产物"，Phase 4 再做"图片创作/相册制作"。

### Skill / Capability Registry

持续性的能力账本：

| 类型 | 含义 | 举例 |
|------|------|------|
| 内置能力 | 出厂就会 | 文本对话、记忆存取 |
| 学到的能力 | 自我成长获得 | 图片处理、语音转文字 |
| 学习中 | 正在尝试 | WebSocket 实现 |
| 暂搁 | 尝试后放弃 | 视频通话（需外部依赖） |

### 自我成长能力的边界

**develop 能力 ≠ deploy 权力**:
- ✅ **develop**: 研究、写 patch、跑测试、生成提案
- 🔒 **deploy**: 合并、热加载、上线 — 需人类审批

> 详细预研方案见 [T08-2 self-growth-spec.md](phase1/t08-daemon/self-growth-spec.md)

## 十、大脑-小脑双向通信

> 不是单向监控，是双向协作。大脑是思考者，小脑是管家。

### 通信方向

```
小脑 → 大脑:                        大脑 → 小脑:
  ├── 心跳监控 (已有)                  ├── SSE 状态上报
  ├── 上下文注入 (MCP)                 ├── 资源请求
  ├── 触发任务                        ├── 自主活动请求
  └── 约束提示                        └── 紧急信号
```

### 技术路径

| 方向 | 推荐方案 | 理由 |
|------|---------|------|
| 小脑 → 大脑 | MCP 工具写共享状态 → 大脑读取 | 和 Phase 2 Memory MCP 方向一致 |
| 大脑 → 小脑 | SSE 事件流 | Engine 层已支持 `subscribeEvents` |

### 职责划分

| 职责 | 大脑 | 小脑 |
|------|------|------|
| 思考推理 | ✅ | ❌ |
| 主动消息 | 生成内容 | 判断时机 + 触发 |
| 自主学习 | 执行学习 | 安排日程 + 守护 |
| 资源管控 | 上报使用量 | 设上限 + 降级 |
| 进程守护 | ❌ | ✅ |
